Visitor Registration
1. Purpose

The purpose of this feature is to allow staff or security personnel to record visitor information when someone enters the university or office. This helps track who visited, whom they visited, and the reason for the visit.

2. Expected User

The expected users of this feature are:

Security personnel

Receptionist

Administrative staff

They will input visitor information into the system.

3. Main Functionality

The Visitor Registration feature allows staff to enter and store visitor details through a form.

The form collects the following information:

Visitor Name

Address

Contact Number

Purpose of Visit

Person to Visit

Date of Visit

Time of Visit

After filling out the form, the information is saved in the system for monitoring and record keeping.

4. Acceptance Criteria

The system allows the user to input visitor information in the registration form.

The system requires all necessary fields (visitor name, contact number, and purpose of visit) before submission.

The system records the date and time of the visit when the form is submitted.

The system successfully saves the visitor information in the database.

The system displays a confirmation message after successful registration.
